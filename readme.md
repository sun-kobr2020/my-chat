У меня есть веб-мессенджер (SPA), собранный на Vite (JS, HTML5, CSS3) и работающий с Firebase 9+ (модульный SDK). 
Проект развернут на GitHub Pages.

### Текущая архитектура проекта:
1. index.html — двухпанельный интерфейс (Сайдбар слева со списком комнат, чат справа). Главный скрипт подключен через 
<script type="module" src="/app.js"></script>.
2. style.css — стили сетки (Flexbox/Grid), стилизация чат-зоны и элементов ввода.
3. vite.config.mjs — конфигурация сборщика Vite с указанием базового пути: base: '/my-chat/'.
4. .env / .env.example — скрытые переменные окружения фронтенда (префикс VITE_FIREBASE_...). Данные вшиваются Vite при сборке. 
В Git файл .env скрыт через .gitignore.

### Стек и База Данных (Firebase Firestore):
- Сообщения хранятся в подколлекциях по пути: rooms / {currentRoomId} / messages / {messageId}
- Список активных комнат хранится в корневой коллекции: all_rooms / {roomId}
- Переключение между комнатами реализовано через URL-параметры: window.location.search = `?room=${randomRoomId}`. 
Если параметра нет, по умолчанию открывается комната 'general'.
- Скрипт слушает изменения БД в реальном времени через onSnapshot.
- Правила безопасности Firestore разрешают чтение и запись только по точным путям комнат (коллекция all_rooms и подколлекции messages). Открыты до июня 2026 года.

### Важные ограничения проекта (Учитывать при доработке!):
1. Firebase Storage НЕ ПОДКЛЮЧЕНО и подключаться не будет (из-за региональных ограничений/ошибок создания бакета Google).
2. Функция handleFileUpload(e) сейчас работает как визуальная заглушка: при выборе файла она блокирует инпут на 2 секунды, 
меняет плейсхолдер на предупреждение об отключенной отправке файлов, а затем возвращает инпут в исходное состояние, НЕ совершая сетевых запросов.
3. Функция отправки текстовых сообщений sendMessage() использует строгий блок try-catch-finally с блокировкой 
sendBtn.disabled = true во время ожидания await addDoc, чтобы избежать спама. Кнопка гарантированно разблокируется в блоке finally.
4. Весь деплой на GitHub Pages автоматизирован через скрипт в package.json: "deploy": "gh-pages -d dist". Сборка запускается через 
npm run build && npm run deploy.
